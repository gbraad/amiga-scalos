Pattern Preferences

With this program you can configure background pictures for the screen, desktop and windows. It consists of two pages for your configuration.

NOTE: If problems should occurr with background pictures, please see Frequently Asked Questions section.

Please ensure you have no Workbench backdrop pattern selected in the original WBPattern preferences program. This is because Scalos uses its own pattern backdrop system, the patterns for the old Workbench will still be loaded but not seen by you, thus wasting your valuable memory.


Patternlist Page

The first allows you to compile a list of pictures, assigning a number to one or more of them for easy reference. Using this number you will be able to assign the pictures to specific windows on the Defaults Page. If multiple pictures have the same number, one of the pictures will be choosen randomly. This will allow you to have random dekstop pictures, random window backdrops etc. without the need for additional utilities/patches in your startup-sequence!



  • Pattern List [listview]
    Pictures added to the datatbase are listed with their assigned number and file path & name.

  • Pattern Viewer [button]
    Clicking on the button will load and display a portion of the currently highlighted pattern in the listview tothe buttons left. Expanding the window size will reveal more of the picture. Also use of the re-size bar inbetween the listview and pattern preview enables adjusting of the width.

  • New/Delete pattern [buttons]
    Add a new picture. After that you should assign a number to it.

  • Tiled/Fit Size [cycle menu]
    Tiled: The picture will be rendered as tiles. (Like original Workbench does)
    Fit-Size:The picture will be resized to the dimension of the new window.You MUST have set GUIGfx check box 'on'.

  • No remap [check box]
    The picture's colors will not be remapped to the actual Screen colors.

  • Use GUIGfx [check box]
    If this option is activated, Scalos uses the math algorithms of the GUIGfx.library. This means fast and high quality remapping to Screen dimensions and colors in all resolutions.

  • Pen Precision [slider]
    Indicates the precision to be use by Scalos in assigning the color pens.

  • GUIGfx Options
    "Use GUIGfx" check box has to be enabled to use these additional options..
    • AutoDither [check box]
      If the picture loses too many colors in the remapping process, some colors will be dithered.

    • Colorweight [slider]
      How many color pens is the picture allowed to set measured as a percentage of the total available pens for the screen.

    • Dithering [cycle]
      Sets the Dithermode to None, Floyd Steinberg or Random.

    • Ditheramount [slider[
      Sets the weight of the dither (only when above Dithering cycle is set to "Random").


Defaults Page

Here you can set the defaults for the background pictures in Scalos.



  • Asyncron layout [check box]
    Pictures will be loaded and rendered while the windows opens (Like original Workbench). If this function is 'off', pictures will always be loaded before opening the windows.

  • Use friendbitmap [check box]
    The picture is present twice in memory for best speed. This option has no effect if V43 picture.datatype or GUIGfx are used.

  • Always relayout [check box]
    If "Fit size" is set, the picture will be scaled everytime the window's dimensions change.

  • Randomize everytime [check box]
    Usually Pictures with the same number will be randomly selected as soon as the configuration loads. If this option is set, the picture will be selected as soon as a window with the same number assignement is opened.

  • Asyncron-Task priority [slider]
    You can set the CPU priority for the Task if "asyncron layout" is set to 'on'.

  • Desktop [slider]
    Number of the Picture for the main window.

  • Screen [slider]
    Number of the picture for the Scalos-Screen.

  • Window [slider]
    Number of the picture for the Scalos-Windows. You may also set a specific pattern for a drawers contents using the     tooltypes facilities.

  • TextMode [slider]
    Number of the picture for the Scalos-Windows in Text Mode.

Tooltypes

The Pattern Prefs program recognises the following additional tooltypes, as well as the standard preferences tooltypes (CREATEICONS, ACTION).

  • NOPREVIEW
    No pattern image preview will be displayed to the right of the pattern list page.

  • AUTOPREVIEW
    Clicking on a picture a preview will be displayed automatically, obviously this has effect only if NOPREVIEW is not set.
