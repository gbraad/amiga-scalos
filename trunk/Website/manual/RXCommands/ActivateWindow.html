Arexx Command

NAME

ACTIVATEWINDOW

SYNOPSIS

ACTIVATEWINDOW [WINDOW] <ROOT|Drawer name>

FUNCTION

This function will make a specified window the currently active window on the desktop. An active window is generally used by other arexx commands to work on. So it is usually required to use this command to get the correct window activated before calling anything else which is going to manipulate files etc.

INPUTS

WINDOW

Either ROOT to activate the desktop window (where volume icons, AppIcons, and left-out icons live) or the fully qualified name of a drawer window to activate which must be already open.

RESULTS

-

EXAMPLE

/* Activate the root window */
ADDRESS workbench
ACTIVATEWINDOW root

/* Activate the "Work:" partition's window if already open*/
ACTIVATEWINDOW "Work:"

ERRORS

  • 10 - If the named window cannot be activated. Can occur if window is not already opened on the desktop.

NOTES

You can only activate windows which belong to Scalos, windows which are showing directory contents as icons or text.

If you choose to have a window activated that is not the desktop window you must make sure that the window name is given as a valid filesystem path name. For example "Work:" is a fully qualified name, and so is "SYS:Devs/Monitors". "Prefs/Env-Archive" would not be a valid window name. A valid name always contains the name of a volume, a device, or an assignment.

BUGS

None
