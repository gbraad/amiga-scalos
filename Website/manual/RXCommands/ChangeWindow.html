Arexx Command

NAME

CHANGEWINDOW

SYNOPSIS

CHANGEWINDOW [WINDOW] <ROOT|ACTIVE|Drawer name> [[LEFTEDGE] <new left edge position>][[TOPEDGE] <new top edge position>] [[WIDTH] <new window width>] [[HEIGHT] <new window height>]

FUNCTION

This function will attempt to alter the size and position of the specified window.

INPUTS

WINDOW

Either "ROOT" to resize/move the Workbench root window (where volume icons and AppIcons live), "ACTIVE" to change the currently active Workbench window or the fully qualified name of a drawer window to change. Note that the drawer window must already be open.

If no WINDOW parameter is specified, this command will try to operate on the currently active Workbench window.

LEFTEDGE New left edge window position. TOPEDGE New top edge window position. WIDTH New window width. HEIGHT New window height.

RESULTS

-

EXAMPLE

/* Change the root window; move it to position 10,30. * and change its size to 200100 pixels */
ADDRESS workbench
CHANGEWINDOW root LEFTEDGE 10 TOPEDGE 30 WIDTH 200 HEIGHT 100

/* Change the currently active window */
CHANGEWINDOW active 20 40 200 100

ERRORS

  • 10 - If the named window cannot be changed; this can also happen if you specified "ACTIVE" as the window name and none of the Workbench windows is currently active.

NOTES

You can only activate windows which belong to Scalos, windows which are showing directory contents as icons or text.

If you choose to have a window activated that is not the desktop window you must make sure that the window name is given as a valid filesystem path name. For example "Work:" is a fully qualified name, and so is "SYS:Devs/Monitors". "Prefs/Env-Archive" would not be a valid window name. A valid name always contains the name of a volume, a device, or an assignment.

BUGS

None
