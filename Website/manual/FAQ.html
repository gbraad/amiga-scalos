1. Installation Problems

1.1 After Installing Scalos and rebooting I no longer have any pull-down menus.

Load up the Scalos Menu Preferences program. You will need to add your menus and then save. Scalos is provided with English menus as standard, the configuration file can be found in the Scalos distribution "Presets/Scalos/Menu_UK.pre" . If you installed Scalos using the provided installer it should have copied the English default menu into "ENVARC:Scalos/Menu.prefs" for you. Additional languages are available as seperate downloads from the Scalos website.


1.2 Scalos displays "Couldn't open iconobject.library" when the library exists in the Libs: directory.

Supplied Datatypes may be not correctly installed. You should test if:

  • "amigaiconobject.datatype" and "iconobject.datatype" exist in SYS:Classes/Datatypes/ ?
  • "Classes/Datatypes" correctly assigned to Libs:?
  • "AmigaIconObject" in "Devs:Datatypes/" (Multiassigns not accepted!) ?
  • mcpgfx.library is installed in your LIBS: directory?


1.3 After installation of Scalos MCP's SysIHack doesn't work correctly.

You need a newer version of MCP. The installed mcpgfx.library is not compatible with MCP version V1.30 or less.
Solution: install again old mcpgfx.library or look for newer MCP version.


1.4 My icons still have borders after i selected no icon frame in the Scalos Prefs!

Have you selected no frames in the main Scalos preferences (Icons page)?
Solution: Ensure you have removed the "Run >NIL: NewIcons" from your s:startup-sequence or s:user-startup.


1.5 Before I installed Scalos I used the DefIcons package, now my setup no longer works.

Solution: Go to the scalos.co.uk downloads page and get the "FileTypes.plugin" package which is a DefIcons compatible package for Scalos.


1.6 What programs aren't compatible with Scalos?

MagicWB-Daemon, ReKeyIt, PatchWB, NewIcons Patch, CommandList

OS3.5 specifics such as AREXX scripts which alter the Workbench require the installation of the wbrexx.plugin available from the Scalos website downloads section.
OS3.5 Programs TweakWB, RAWBInfo (unless you delete 'scalos:modules/information.module')




2. Graphical Problems

2.1 No background pictures loaded, though they are correctly defined in the Pattern Preferences.

This may be caused by old Datatypes. Try to reinstall the Datatypes from the Workbench disk, or the actual version of V43 picture.datatypes. You will find many new and optimised datatypes in the Aminet archive.


2.2 Many of my Icons display more than once on the Scalos desktop, while on the Workbench all seems ok.

The Workbench filters double displayed icons, Scalos does not.
Solution: please edit the ".backdrop" file using a standard ASCII editor, and clear lines containing double entires.


2.3 Background images not scaled.

GUIGfx option not set in the Scalos Pattern Preferences or guigfx.library and/or render.library are not installed in your LIBS: directory. You may obtain the latest versions of the two libraries from the Aminet archives.


2.4 Standard NewIcons display with borders.

This may be caused by a missing function of the newicon.library in NewIcons V3. You should use at least NewIcons V4 (which was provided in the Scalos installation).


2.5 Left out icons (on the Scalos desktop) or programs (AppIcons) aren't transparent.

That's because of conficts with the newicons patch.
Solution: You should remove the line "Run >NIL: NewIcons" from your s:startup-sequence or s:user-startup


2.6 Problems with CyberGraphX 3 or 4 and the 'BitmapCache=ON' tooltype.

The BITMAPCACHE tooltype of CybergraphX drivers is usually reuqired for the Warp3D system to function at all.
Sometimes NewIcons will be displayed trashed if the BitmapCache feature is enabled in your monitor drivers tooltypes. This bug is fixed in CyberGraphX v4.2+ which can be downloaded as an upgrade from the commercial v4 CD-ROM from http://www.vgr.com/V4


2.7 The Scalos desktop backdrop image doesn't fit the screen correctly, bits seem to hang off the edges of the screen, and it tiles the picture even though i specified no tiling in the pattern prefs!

Solution: First make the desktop into a window by unselecting the 'Backdrop' option for the first menu (just like you can do on the old Workbench). Make this main desktop window to the full size of the screen, then snapshot the window using the usual option in the 'Window' menu. Now set the desktop window back to 'Backdrop' as usual.



3. System Problems

3.1 Working with CD's causes crashes or Scalos doesn't function correctly when accessing some file systems.

Most Filesystems don't support the ExAll function correctly. Disable "Use ExAll" in the Scalos Main preferences.


3.2 Scalos is using lots of chip ram for every directory window I open.

Scalos needs, normally, more chip ram than the original WorkBench desktop. However IPrefs can be causing some memory wastage as it is still loading window patterns too (as used by the original Workbench).

Scalos uses multiple tasks when working, a window is a seperate task from the desktop - hence the ability to multitask. If usually you have all graphics in fast RAM using Fblit when using Workbench you will need to configure the FBlit patch to also work with Scalos.

Solution: Remove all pictures in 'WBPattern' preferences.
Ensure both 'Scalos_Window_Task' and 'Workbench' are in the include list for FBlit.


3.3 Scalos doesn't start any programs from the WBStartup drawer.

'WBStartup Path' may be set wrongly. Use the Scalos Main Preferences program to edit the path used by Scalos.


3.4 Scalos crashes while starting the WBStartup programs.

Maybe you've got the program "Cyberpatcher" in your WBStartup drawer? Please remove it from there and start it in the s:user-startup.


3.5 Using OS3.9 and "Icon Information" from the menu it appears that some tooltypes are not being saved. Sometimes even the icons are no longer recognised.

Delete the 'Scalos:Modules/information.module' and Scalos will fall back to using the workbench.library standard version. With OS3.5+ this could be "RAWBInfo" add-on which works very well.
